# builders-radar Report Style

Follow these style rules unless the input explicitly requires otherwise:

- Use a calm, analytical tone
- Prefer product and engineering angles over hype
- Be selective: do not include an item only to make coverage look broad
- Avoid filler such as “值得一提的是”, “可以看到”, “总的来说” unless it adds real value
- Avoid bullet spam inside each item; short paragraphs are preferred
- If a tweet or post is trivial, skip it or collapse it into a brief note
- Keep empty sections to one short plain statement when there is no meaningful update
- When summarizing opinions, separate observation from implication
- Keep the report readable in GitHub Pages on both desktop and mobile
