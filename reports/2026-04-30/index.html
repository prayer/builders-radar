Josh Woodward 推出 Gemini 的文件生成与导出能力

Josh Woodward 宣布 Gemini 现在可以直接生成并导出文件，覆盖 Google Docs、Word、PDF、Sheets、Excel、CSV、Slides，以及 Markdown、LaTeX、TXT、RTF。这个更新的重点不是聊天本身，而是把生成结果直接落到可交付文档和表格上，减少从对话到成品的摩擦。

  • https://x.com/joshwoodward/status/2049524400131838436

Aaron Levie 认为企业会出现专门的 agent engineering 角色

Aaron Levie 说，Box 正在招聘和训练面向内部功能的 agent engineering 角色，目标是把 agents 接到 Box、Salesforce、Workday 等业务系统里，并把流程编码成 skills。核心判断是，未来的自动化重点不是单个岗位，而是跨团队、跨系统的业务流程。

  • https://x.com/levie/status/2049714403050918067

Aditya Agarwal 把当前 agents 定位为开发者产品

Aditya Agarwal 总结自己在 OpenClaw、Hermes 等工具上的使用体验后认为，今天的 agents 还远不是消费级产品，可靠性和易用性都不够。他更倾向于把它们理解为代码生成和 tool-calling 的包装层，而不是“生成式 AI”本身，同时他明确期待更动态的界面，而不是继续停留在 chat。

  • https://x.com/adityaag/status/2049530878083379427

Sam Altman 把 GPT-5.5-Cyber 推向关键网络防御场景

Sam Altman 表示，OpenAI 正在向关键 cyber defenders 分批 rollout GPT-5.5-Cyber，并会和生态与政府一起讨论可信访问机制。这个动作说明 frontier model 正在更明确地进入安全防护链路，而不只是通用助手。

  • https://x.com/sama/status/2049712078836170843

Swyx 强调应该更多实验 base model 的奇怪用法和 fine-tunes

Swyx 表示，他对我们终于拿到一个 base model 这件事本身就很兴奋，也认为围绕现有 base models 的非常规用法和 fine-tunes 还远远不够。整体语气是：与其只讨论“completions API 是否过时”，不如把精力放到把现有模型用出新东西。

  • https://x.com/swyx/status/2049643473339601388

Garry Tan 用 E2E harness 保障 gbrain 安装和 onboarding

Garry Tan 说他终于给 openclaw 上的 gbrain 安装流程做了完整的 E2E harness，这样可以确认 onboarding 过程真的可用。另一个更新是，graphs 正在更多依赖 frontmatter attributes，而且越来越多内容会被推断和自动构建，说明他的系统在向更强的结构化和自动化演进。

  • https://x.com/garrytan/status/2049742376307147110
  • https://x.com/garrytan/status/2049737217208397925

Claude 展示 Managed Agents 的两个落地方向

Claude 账号今天展示了两种 Managed Agents 用法：ARIA 是读取机器手册、出故障后自动生成工单的维护系统，MaestrIA 是拍摄家修损坏、返回诊断、给本地商家估价并草拟联系维修人员消息的工具。两者都指向同一个结论：agents 的早期高价值场景更接近维护、报修和流程编排，而不是纯对话。

今天有一档高信号播客，核心是 Stripe 对 agent-native 经济的系统判断。

AI & I by Every: Stripe 看见的 agent-native 世界

这期的主线很清楚：Internet 的主要参与者正在从人转向 agents，因此支付、风控、计费和开发者基础设施都要重做。Emily Glassberg Sands 认为，fraud 已经从交易层问题变成全漏斗问题，因为 AI 公司最容易被盗的资产不再只是卡信息，而是 compute、free trial、credit 和后付账期；Stripe 的 Radar 也因此从 checkout 往 signup、overage 和 dispute 延伸，并且开始覆盖更多 payment methods 和外部 processor。定价层面，她判断模型提供方会继续按 token 计费，垂直 AI 产品更可能走 outcome-based pricing，而 seat-based billing 在企业里会快速失去合理性。对 agentic commerce，她没有把它描述成“全自动代买”，而是拆成从 AI 辅助搜索，到受限委托购买，再到更远期的 ambient agent；对应的底层原语是 catalog、shared payment token、merchant of record 和 fraud scores。
