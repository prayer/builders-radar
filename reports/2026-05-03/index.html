这一期播客的主轴很清晰：AI 时代最稀缺的资源不是算力本身，而是人类注意力、上下文和对高风险动作的判断。

Training Data: OpenAI 的 Greg Brockman 谈“注意力瓶颈”与 agent 化工作流

这一期最重要的判断是，AI 的瓶颈正在从“模型能不能做”转向“人类是否有精力提供足够上下文、审批和治理”。Greg Brockman 先从算力讲起，把 OpenAI 的业务概括为“买、租、建 compute，再以正毛利转售”，并明确说算力永远不够，需求增长会持续吃掉供给。更值得注意的是，他把 scaling laws 描述为一种经验上仍在展开的“深层谜题”，同时强调模型进步往往来自细小但关键的工程和数据处理改动，而不只是更大的模型。对创业者的建议也很直接：继续向 agentic 工具倾斜，因为过去几个月它们已经从“写 20% 代码”走到“写 80% 代码”，而真正的拐点在于让 AI 拿到足够上下文、让它能跨会议和跨系统理解任务。OpenAI 内部的实践则显示，应用层的重点正在变成治理、权限、可观测性和责任边界，比如保留“人类对合并代码负责”的要求。安全方面，他认为模型可以用于扫描代码库和端到端红队，但前提是配合 trusted access programs 和更严格的风险分层。

官方博客今天的重点是长程 agent 基础设施怎么设计，核心不是单个模型能力，而是把会变化的实现和必须稳定的接口分开。

Anthropic Engineering：Managed Agents 把“brain”和“hands”解耦

Anthropic 发布 Managed Agents，核心思路是把 agent 的 session、harness 和 sandbox 分离，避免把长程执行绑死在单一容器或单一实现上。文章最实用的部分在于，它明确指出 harness 里很多“模型做不到”的假设会随着模型进步迅速过时，例如以前需要上下文重置来应对“context anxiety”，到了更强的模型上反而成了冗余逻辑。架构上，session 作为持久化事件日志放在上下文窗口之外，sandbox 负责执行，harness 通过接口调度两者，这样既提高可恢复性，也把凭据隔离到 sandbox 外部，降低 prompt injection 风险。Anthropic 还给出性能结果：这种解耦让 p50 TTFT 下降约 60%，p95 下降超过 90%。对构建 agent 平台的人来说，这篇文章的结论很明确：接口要比当前实现更长寿，状态、计算和权限必须拆开设计。
