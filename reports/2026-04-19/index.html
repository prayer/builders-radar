今天的官方博客更新主要来自 Anthropic，分别对应评估方法论和 Claude Code 权限机制。

Anthropic Engineering：agentic coding eval 的分数会被基础设施配置显著扰动

这篇文章的结论很直接：在 Terminal-Bench 2.0 和 SWE-bench 这类 agentic coding eval 里，硬件和 sandbox 配置本身就能带来足以改变排名解释的分数差异。Anthropic 发现，在 Terminal-Bench 2.0 上，从严格 1x 限制到更宽松资源头寸，infra error 会明显下降，而到 uncapped 时总体 lift 可达 6 个百分点；但 1x 到 3x 之间的分数变化仍大多落在噪声范围内。对 builder 的实际意义是，leaderboard 上几分之差不能脱离资源配置单独解读，至少要把 CPU、RAM、limit 和 enforcement method 一起报告。

  • https://www.anthropic.com/engineering/infrastructure-noise

Anthropic Engineering：Claude Code auto mode 用分类器替代部分权限确认

这篇文章在解决一个现实问题：手动 approval 太容易产生 fatigue，而直接跳过权限又太危险。auto mode 通过输入层的 prompt-injection probe 和输出层的 transcript classifier 两道防线，把原本需要人工确认的高风险动作交给模型化判定，并用两阶段分类平衡误报和漏报；公开数据里，部署后的 FPR 降到 0.4%，但对真实 overeager actions 的 FNR 仍有 17%。它不是高风险基础设施的人工审查替代品，但确实提供了一个比 `--dangerously-skip-permissions` 更安全、也更低摩擦的中间态。

  • https://www.anthropic.com/engineering/claude-code-auto-mode
