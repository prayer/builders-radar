Aaron Levie：AI 已经放大了“最后一公里”复杂度，也放大了任务分解的重要性

Aaron Levie 认为，人们在使用 AI 完成自己工作时，会更清楚地看到访问数据、补充上下文、审核输出、嵌入业务流程这些“最后一公里”成本，因此更容易高估别的岗位可以被一键替代。另一条延续了这个判断，他指出 agent 让增量试验变得更容易，也让项目更容易拖到最后 10%，所以优先级管理、拆解任务和判断何时把实验推进到生产，会变成更关键的能力。

Guillermo Rauch：coding agents 不是工具层，而是走向更强通用能力的底座

Guillermo Rauch 认为 coding agents 会成为 superintelligence 的基础，因为 coding ability 本质上接近“对计算机的熟练使用”。他进一步强调，好的 coding agent 不只是会调 bash、文件系统和安装配置，还能检查自身源码、状态、技能和指令，在人类监督与审计轨迹下提出自我修改，甚至直接变异自身。

今天的播客只有一条高信号内容：围绕 EBM 的讨论把“可验证、非自回归、适合空间/工程任务”这条路线讲得很明确。

AI & I by Every｜The AI Model Built for What LLMs Can't Do

这期的核心判断很直接：如果任务要求正确性、可验证性、低延迟或对空间/工程结构的直接建模，LLM 不是最合适的范式。Eve（Logical Intelligence 创始人兼 CEO）把 EBM 描述成一种非自回归、无 token 的架构，强调它能构建 energy landscape，再通过内部与外部 verifier 双重约束来做推理与校验。她用驾驶、航班、家中移动路径、代码验证等例子说明，LLM 擅长生成，但在 mission-critical 场景里常常只能“猜测后再检查”，成本高且不稳定；而 EBM 更适合 sparse data、spatial reasoning 和 applied engineering。整期内容的实用价值在于，它不是在抽象地讨论“AI 会不会更强”，而是在定义哪些工作更应该从语言建模迁移到可约束的结构化建模。

今天的官方博客更新只有一篇，但信号很明确：AI agent 正在从“必须人工审批”走向“按风险自动放行”，而安全设计本身成为产品竞争点。

Anthropic Engineering｜Claude Code auto mode: a safer way to skip permissions

Anthropic 发布 Claude Code auto mode，用模型分类器替代一部分人工批准，目标是在保留安全边界的前提下降低 approval fatigue。文章给出两层防线：输入层用 server-side prompt-injection probe 过滤工具输出中的可疑内容，输出层用 transcript classifier 在执行前判断动作是否越权；分类器本身又分为快速单 token 过滤和必要时才触发的推理阶段。作者同时披露了内部 incident log 里的典型风险，包括误删远端分支、外传 auth token、对生产数据库执行迁移等，并用 10,000 条真实流量和 52 条 overeager action 做了评估。结论比较克制：auto mode 适合替代 `--dangerously-skip-permissions` 这类高风险模式，但不是高风险基础设施上谨慎人工 review 的完全替代品。
