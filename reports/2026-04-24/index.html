今天 X 上最集中的信号仍然是 GPT-5.5、Codex 和 Claude Code 周边的产品反馈，以及 agents 的工程化落地；同时，memory、connector、eval 和检索栈开始被当成真正的生产基础设施。

Aaron Levie（Box CEO）：GPT-5.5 对企业知识工作有明显提升

他用 Box 的企业知识工作评测观察到，GPT-5.5 相比 GPT-5.4 的准确率提升约 10 个百分点，金融、医疗、公共部门和传媒场景都更强。更重要的判断是，AI 不是让工作变少，而是把原本做不到的项目变得值得启动，进而把团队的工作量推高。对企业 agent 来说，价值已经从“能不能做”转到“能不能把长链路任务稳定做完”。

podcasts

Podcasts

本期 Jacob Efron 与 Latent Space 的 Swyx 讨论的核心判断是：2026 更像是 coding agents 从代码场景破圈去做其他事的一年，而不是单纯追求更高效率的一年。嘉宾认为，AI infra 正在围绕 skills、harness、context engineering、evals 和 observability 形成相对稳定的工程栈，但 memory、personalization、实时能力和 subagents 仍会继续改变产品形态。市场结构上，他倾向于两大 coding 平台加少量长尾，且企业更需要能补齐最后一公里的 agent lab；他也明显转向更看好 open models、RL 和 synthetic rubrics，认为真正的分水岭会出现在多轮定制、低成本高频实验和 zero human review 式的工作流。

Unsupervised Learning Ep 85：AI infra 是否已经稳定、FM 叙事如何变化、coding agents 下一步是什么

本期把 coding wars、infra 稳定化、open models 和 2026 的 agent 走向放在一起看，结论偏向能力继续外扩，而不是回到收缩和效率化。Swyx 认为，2025 是 coding agents 之年，2026 会进入破圈阶段，核心变化不再是单点模型能力，而是记忆、个性化、子 agent、低成本实验和生产级 harness 如何组合成真正可持续的工作流。市场结构上，他更看好两大平台加长尾的格局，也更愿意把 open models、RL、synthetic rubrics 和 dark factory 式工作流视为下一轮值得下注的方向。

Blogs

官方博客最重要的是 Claude Code 回退问题的技术复盘、生活类 connectors 的扩展，以及 Managed Agents memory 的 public beta。三篇文章共同强调了一点：面向生产的 agent 产品，关键不只是模型能力，而是可控、可审计、可回滚的产品层设计。

Anthropic Engineering：Claude Code 质量回退的根因是三处产品层改动

这篇复盘把近期“回答变差”的反馈拆成了三条独立问题：Claude Code 默认 reasoning effort 从 high 改到 medium、idle 会话的 thinking 清理 bug、以及一个削弱 verbosity 的 system prompt 变更。关键结论是 API 和推理层没有受影响，问题出在产品层组合效应；他们已在 4 月 20 日前后全部修复，并开始重设使用额度。更重要的是，这篇复盘暴露了内部 code review、eval 和 dogfooding 仍然可能漏掉 corner case，因此后续会加更广的 per-model eval、prompt 审计和更严格的 rollout 流程。

  • https://www.anthropic.com/engineering/april-23-postmortem

Claude Blog：Claude 把生活类 connectors 扩到更多日常应用

Claude 现在不只接工作软件，也开始连接 AllTrails、Instacart、Audible、TripAdvisor、TurboTax、Uber、Resy 等日常服务。产品重点不是多接几个 app，而是让 connectors 在对话里按任务动态浮现，并在预订或购买前显式确认，保持无广告、不过度共享数据、可随时断开。对 builders 来说，这说明 Claude 正把 connector directory 变成新的分发面和行动入口。

  • https://claude.com/blog/connectors-for-everyday-life

Claude Blog：Claude Managed Agents 的 Memory 进入 public beta

Managed Agents 现在支持跨 session memory，记忆以文件形式存储，能通过 API 导出、共享、回滚和审计。Anthropic 强调它适合生产环境，因为 memory 直接挂在 filesystem 上，并支持作用域隔离、并发访问和可追踪的 session event。案例里，Netflix、Rakuten、Wisedocs 和 Ando 都在用它减少重复错误、加速验证或替代自建检索层，说明 memory 正在从附加功能变成 agent 基础设施。

  • https://claude.com/blog/claude-managed-agents-memory
